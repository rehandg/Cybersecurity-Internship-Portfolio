# TASK_5--Wireshark
Wireshark Network Traffic Analysis Report
To capture and analyze live network traffic using Wireshark, identify at least three network protocols, and compare normal browsing behavior with traffic generated by an Nmap port scan and browsing.

Tools Used
Wireshark vX.X.X

Nmap v7.94 (for scanning)

Operating System: Kali Linux / Windows 10 (specify your environment)

Network Interface Used: eth0 / wlan0 / Wi-Fi (choose the relevant interface)

Test 1: Normal Browsing Traffic
Traffic Generation
Opened a browser and visited example.com and google.com

Capture duration: approximately 1 minute

Captured Protocols
DNS (Domain Name System)

Function: Resolves domain names to IP addresses

Observed: Queries for example.com, google.com

Packet Example:

Source Port: 54621

Destination Port: 53

Query Type: A (IPv4 address)

HTTP (HyperText Transfer Protocol)

Function: Unencrypted web traffic

Observed: GET requests to example.com

Packet Example:

Method: GET

Host: example.com

User-Agent: Mozilla/5.0

TCP (Transmission Control Protocol)

Function: Ensures reliable, ordered delivery of packets

Observed: TCP handshakes between client and remote servers (port 80, 443)

Packet Example:

Flags: SYN, SYN-ACK, ACK

Src Port: 54622

Dst Port: 443

Test 2: Nmap Port Scanning Traffic
Traffic Generation
Ran the command: nmap -sS 192.168.1.1

Capture duration: approximately 1 minute

Additional Captured Protocols
TCP SYN Scanning

Observed: Many SYN packets sent to sequential ports on target IP

Details: No full handshake, only SYN packets used for port discovery

Packet Example:

Flags: SYN only

Dst Ports: 22, 80, 443, 445, 3389, etc.

ARP (Address Resolution Protocol)

Observed: ARP requests issued to find the MAC address of target IP before scanning

Packet Example:

Who has 192.168.1.1? Tell 192.168.1.100

ICMP (Internet Control Message Protocol) (Optional)

Observed: If a ping sweep was performed or host discovery included

Packet Example:

Type: 8 (Echo Request)

Code: 0

Summary of Protocols Identified
Protocol	Function	Found in
DNS	Name resolution	Normal Traffic
HTTP	Web browsing	Normal Traffic
TCP	Reliable transport	Both
ARP	MAC address resolution	Nmap Scan
ICMP	Ping echo requests	Nmap Scan (optional)

Files Attached
normal_traffic.pcap – capture of normal browsing

nmap_scan_traffic.pcap – capture during Nmap scanning

Conclusion
Wireshark successfully captured both routine and scanning-related network traffic. During normal browsing, DNS, HTTP, and TCP protocols were primarily used. The Nmap scan introduced distinct SYN-only TCP packets and ARP queries, which are indicative of reconnaissance activity. This comparison shows how different types of traffic can be identified and analyzed using packet capture tools.
